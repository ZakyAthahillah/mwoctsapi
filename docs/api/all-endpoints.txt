Access Notes
------------
- Public without token:
  POST /api/register
  POST /api/login
  POST /api/refresh

- Require JWT token:
  POST /api/logout
  GET /api/me
  GET /api/profile
  POST /api/profile
  GET /api/dashboard
  All downtime endpoints
  All reporting endpoints
  All reporting report endpoints
  All job endpoints
  All MTBF endpoints
  All MTTR endpoints
  All FBDT endpoints
  All target endpoints
  All area endpoints
  All division endpoints
  All group endpoints
  All informant endpoints
  All machine endpoints
  All operation endpoints
  All part endpoints
  All position endpoints
  All reason endpoints
  All serial number endpoints
  All shift endpoints
  All technician endpoints

- Require JWT token and admin middleware:
  All role endpoints
  All permission endpoints
  GET /api/users
  PUT /api/users/{user}
  DELETE /api/users/{user}

Positions Examples
------------------
List position_active
GET /api/position_active?per_page=10&search=posisi

Toggle position status 99 <-> 1
PUT /api/position_setstatus/{position}

List positions
GET /api/positions?per_page=10&search=posisi

Create position
POST /api/positions
{
  "name": "Posisi A",
  "description": "Deskripsi posisi",
  "status": 1
}

Update position
PUT /api/positions/{position}
{
  "area_id": 1,
  "name": "Posisi A Update",
  "description": "Deskripsi posisi update",
  "status": 1
}

Delete position
DELETE /api/positions/{position}

Common Headers
--------------
Use these headers for authenticated endpoints:

Authorization: Bearer <your_jwt_token>
Accept: application/json
Content-Type: application/json

Machines Examples
-----------------
List machine_active
GET /api/machine_active?per_page=10&search=mesin
Returns total_part and total_position for each machine.

Toggle machine status 99 <-> 1
PUT /api/machine_setstatus/{machine}

List machines
GET /api/machines?per_page=10&search=mesin
Returns machines with status 1 and 99.
Returns total_part and total_position for each machine.

List machine options for legacy selector
GET /api/machines/full-data-array?term=mesin

List active machine options for job selector
GET /api/machines/full-data-array-job?term=mesin

Get machine detail with mapped parts
GET /api/machines/{machine}/detail

Returned detail includes front pins in data.parts and side pins in data.parts_side.
Returned detail also includes data.position_id, data.position_name, data.total_part, and data.total_position.

Get machine positions
GET /api/machines/{machine}/positions?term=front&selected=2

Get machine job detail for a position
GET /api/machines/{machineId}/positions/{positionId}/detail-job

Get machine parts for a position
GET /api/machines/{machineId}/positions/{positionId}/parts

Activate or deactivate completed machine
PUT /api/machines/{machine}/activate
{
  "Authorization": "Bearer <your_jwt_token>"
}

Create machine
POST /api/machines
{
  "code": "MCH001",
  "name": "Mesin Potong",
  "description": "Mesin untuk proses potong",
  "image": null,
  "image_side": null,
  "position_id": [1, 2],
  "status": 1
}

Create machine with default status and nullable images
POST /api/machines
{
  "code": "MCH002",
  "name": "Mesin Default",
  "description": "Mesin tanpa image",
  "position_id": [1, 2]
}

Saved values example for string input:
image=machines/{id}/front.png
image_side=machines/{id}/side.png

Create machine with file upload
POST /api/machines
Headers:
Authorization: Bearer <your_jwt_token>
Accept: application/json
Content-Type: multipart/form-data

Form fields:
code=MCH001
name=Mesin Potong
description=Mesin untuk proses potong
image=<front-image-file>
image_side=<side-image-file>
position_id[]=1
position_id[]=2
status=1

Saved values example:
image=machines/10075/front.jpeg
image_side=machines/10075/side.jpeg

Update machine
PUT /api/machines/{machine}
{
  "area_id": 1,
  "code": "MCH001",
  "name": "Mesin Potong Update",
  "description": "Mesin update",
  "image": "machines/10075/front.png",
  "image_side": "machines/10075/side.png",
  "status": 1,
  "parts": {
    "id": [1, 2],
    "x": [12.4, 55.2],
    "y": [22.1, 66.7],
    "x_side": [10.4, 51.2],
    "y_side": [20.1, 61.7]
  }
}

Update machine without changing pins
PUT /api/machines/{machine}
{
  "area_id": 1,
  "code": "MCH001",
  "name": "Mesin Potong Update",
  "description": "Mesin update",
  "image": "machines/10075/front.png",
  "image_side": "machines/10075/side.png",
  "status": 1
}

Update machine pins using form-data compatible JSON string
PUT /api/machines/{machine}
Form field:
parts={"id":[1,2],"x":[12.4,55.2],"y":[22.1,66.7],"x_side":[10.4,51.2],"y_side":[20.1,61.7]}

Delete machine
DELETE /api/machines/{machine}
